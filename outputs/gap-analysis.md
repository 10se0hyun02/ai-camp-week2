# 초안 ↔ 골든 격차 분석 워크시트

> `outputs/draft-1.md` vs `golden/output-example.md` 비교

---

## 1. 빠진 정보 (골든엔 있는데 초안엔 없는 것)

- 에러별 **발생 횟수** 없음 (E-01이 4회 연속이라는 정보 누락)
- **심각도(HIGH/MEDIUM)** 분류 없음
- **인과관계** 미파악 (W-01 슬로우 쿼리 → E-02 커넥션 풀 소진 연결 없음)
- **배포 버전(v2.3.1)과 E-05의 연관성** 언급 없음
- **액션 아이템 기한·담당팀** 없음
- W-02 RefreshToken 급증이 3배라는 수치 정보 누락

## 2. 구조 차이 (순서·섹션·형식이 다른 부분)

- 골든: 분류표 → 근본원인 → 액션 아이템 → 사람 검토 항목 → 한 줄 요약 순
- 초안: 산문 나열 → 목록 → 액션 3줄로 끝 (섹션 구분 없음)
- 골든은 표(Table) 형식, 초안은 불릿 리스트
- 골든은 에러 ID(E-01, E-02…)로 추적 가능, 초안은 불가

## 3. 톤 차이 (말투·길이·격식이 다른 부분)

- 초안은 "아마 DB 쪽에서 뭔가 잘못된 것 같습니다" → 추측성 구어체
- 골든은 "W-01 슬로우 쿼리(4823ms)가 커넥션을 점유" → 로그 수치 근거 제시
- 초안은 전체 분량이 지나치게 짧음 (액션 아이템 3줄)
- 격식: 초안은 블로그 글투, 골든은 운영 보고서 투

## 4. 사실 오류 / 추측 (초안이 입력에 없는 걸 만들어낸 부분)

- "DB 커넥션 풀 늘리기" → 입력 로그에 없는 해결책을 AI가 임의로 제안
- "외부 API 재시도 로직 추가" → 이미 retry=3/3으로 소진된 상황인데 추가 제안은 맥락 오류
- "시스템 안정성에 문제가 있어 보이며" → 근거 없는 포괄적 판단

## 5. AI↔사람 경계 위반 (사람이 봐야 할 걸 AI가 멋대로 처리한 부분)

- E-04 DataCorruption: 재처리 여부를 AI가 판단하지 않고 그냥 목록에만 넣음 (⚠️ 표시 없음)
- E-05 ClassCastException: 롤백 여부를 AI가 판단해야 할 사항으로 남겨야 하는데 언급 없음
- W-02 토큰 급증: 보안 위협 가능성 → "⚠️ 사람 검토 필요" 표시 없음

## 6. 그 외 어색한 점

- 에러 발생 시간대·범위 요약 없음 (분석 범위 헤더 미포함)
- 서비스별 집계가 없어서 어느 서비스가 가장 문제인지 한눈에 안 보임

## 7. 다음 주에 줄 "제약 조건" 후보

- **출력 구조 고정:** 반드시 [분류표 → 근본원인 → 액션 아이템 → 사람 검토 항목 → 한 줄 요약] 5섹션 순서
- **수치는 입력 그대로:** 발생 횟수·시간·ms·버전 등 로그에 있는 수치는 반올림·생략 없이 그대로
- **추측성 문장 금지:** "아마", "것 같습니다", "보이며" 같은 표현 사용 금지 — 근거 없으면 "원인 확정 불가"로 표기
- **사람 검토 필요 케이스 표시:** 배포 롤백·데이터 재처리·보안 위협 판단은 반드시 "⚠️ 사람 판단 필요"로 분리
- **에러 ID 부여:** 각 에러에 E-NN / W-NN ID를 부여해 액션 아이템과 교차 추적 가능하게

## 한 줄 회고

첫 초안에서 가장 크게 빗나간 한 가지는 **구조와 수치 누락** 이었고,
그 이유는 **(컨텍스트가 부족 / 규칙을 안 줌)** 인 것 같다.
→ 골든 출력 형식을 명시적으로 주지 않으면 AI는 "그냥 요약"을 한다.
