# 2주차 과제: 컨텍스트 세팅 + 첫 격차 분석

## 수강생 정보
- 이름: 조서현
- 직무: DX팀 — 마케팅솔루션 구축 및 운영 담당자
- 자동화 주제: 에러 로그 추적 자동화

---

## 1. 이번 주 산출물 + AI/사람 역할 경계

**산출물:** 마케팅솔루션 에러 로그 일일 요약 리포트 (Slack 공유용)

**AI/사람 역할 경계 (한 줄):**  
AI는 에러 로그를 수집·분류·심각도 라벨링·요약 초안까지 처리하고, 담당자(사람)는 원인 판단·조치 결정·팀 Slack 채널 실제 발송을 담당한다.

## 3. 이상적인 출력 예시 (골든 출력)

> 위 입력 로그에 대해 담당자가 직접 작성한 "이렇게 나오면 완벽하다" 기준의 출력

---

**[DX팀] 마케팅솔루션 에러 리포트 — 2026-05-15 09:00~10:00**

| 심각도 | 서비스 | 에러 코드 | 건수 | 요약 |
|---|---|---|---|---|
| 🔴 긴급 | ga4-connector | 403 | 2건 | GA4 property 권한 만료 — 서비스 계정 재인증 필요 |
| 🔴 긴급 | data-pipeline | 504 | 1건 | BigQuery 쿼리 타임아웃 — 쿼리 최적화 또는 제한시간 상향 필요 |
| 🟡 주의 | crm-sync | 429 | 1건 | CRM API 속도 제한 도달 — 배치 간격 조정 권장 |
| 🟡 주의 | ad-platform-meta | 500 | 1건 | Meta API 서버 오류 (일시적) — 자동 재시도로 해결 예상, 지속 시 Meta 상태페이지 확인 |
| ⬜ 정상 | crm-sync | 409 | 1건 | 중복 연락처 감지 → 스킵 처리됨 (정상 동작) |

**조치 필요 항목 (담당자 확인 요망):**
1. `ga4-connector` 서비스 계정 권한 재발급 (IAM 확인)
2. `bq-job-20260515-0041` 쿼리 원인 파악 후 최적화

**참고:** email-trigger 배치(1,842건 발송)는 정상 완료.

## 4. AI 첫 초안 (시뮬레이션)

> 골든 입력 없이 "에러 로그 요약해줘"만 입력했을 때 AI가 생성한 첫 초안

---

**에러 로그 분석 결과**

오늘 오전 9시~9시 30분 사이에 여러 서비스에서 에러가 발생했습니다.

1. **ga4-connector**: 403 Access Denied 에러가 2번 발생했습니다. 권한 문제로 보입니다.
2. **crm-sync**: 429 Rate Limit 에러와 409 Conflict 에러가 발생했습니다.
3. **ad-platform-meta**: 500 Internal Server Error가 발생했습니다.
4. **data-pipeline**: 504 Timeout 에러가 발생했습니다.
5. **email-trigger**: 정상 완료되었습니다.

위 에러들을 확인하고 적절한 조치를 취하는 것이 좋을 것 같습니다.

## 5. 초안 ↔ 골든 격차 분석 (5가지 기준)

### ① 빠진 정보 (골든엔 있는데 초안엔 없는 것)
- 심각도 구분(긴급/주의/정상)이 없어 무엇을 먼저 처리해야 할지 알 수 없음
- 구체적인 조치 방향(서비스 계정 재인증, 쿼리 최적화 등)이 전혀 없음
- 에러 건수 집계 없음 (ga4가 2번 반복된 것을 식별 못 함)
- email-trigger 정상 완료 수치(1,842건)가 누락됨

### ② 구조 차이 (순서·섹션·형식이 다른 부분)
- 골든 출력은 표 형식으로 한눈에 파악 가능하나 초안은 번호 목록으로만 나열됨
- 골든 출력은 "조치 필요 항목"을 별도 섹션으로 분리했으나 초안은 혼합 서술
- 골든 출력은 서비스명 + 에러코드 + 건수 + 요약이 구조화, 초안은 텍스트 나열

### ③ 톤 차이 (말투·길이·격식이 다른 부분)
- 초안의 "좋을 것 같습니다" → 불필요한 완곡 표현, 운영 리포트엔 적합하지 않음
- 골든 출력은 Slack 공유에 맞는 간결·직접적 말투인 반면 초안은 서술형 보고서 투
- 초안이 더 길지만 정보 밀도는 낮음

### ④ 사실 오류 / 추측 (초안이 입력에 없는 걸 만들어낸 부분)
- "오늘 오전 9시~9시 30분"으로 시간 범위를 임의 축소 (실제는 9:12~9:23)
- "권한 문제로 보입니다" → 403은 명확한 권한 거부이므로 추측 표현 불필요

### ⑤ AI↔사람 경계 위반 (사람이 봐야 할 걸 AI가 멋대로 처리한 부분)
- 초안 자체는 경계 위반 없음 (요약에 그침)
- 그러나 조치 방법을 구체적으로 명시하지 않아 사람이 판단해야 할 포인트를 안내하지 못함
- "적절한 조치를 취하는 것이 좋을 것 같습니다"는 너무 막연하여 담당자 액션을 이끌어내지 못함

## 6. 3주차 제약 조건 후보

> 위 격차를 메우기 위해 에이전트에게 추가로 줄 규칙

1. **심각도 라벨링 필수:** 출력에는 반드시 `🔴 긴급 / 🟡 주의 / ⬜ 정상` 3단계 심각도 라벨을 붙인다. 기준: 5xx·반복 4xx → 긴급, 1회성 4xx·rate limit → 주의, INFO 정상 완료 → 정상.
2. **표 형식 고정:** 출력은 반드시 `심각도 | 서비스 | 에러코드 | 건수 | 요약` 5열 표로 시작하고, 그 아래 "조치 필요 항목" 번호 목록을 붙인다.
3. **추측성 표현 금지:** "~인 것 같습니다", "~보입니다", "좋을 것 같습니다" 등 완곡 표현을 사용하지 않는다. 원인이 불명확하면 "원인 미상 — 추가 확인 필요"로 명시한다.
4. **반복 에러 집계 필수:** 동일 service + code 조합이 2건 이상이면 건수를 집계하여 표시한다. 반복 에러는 `🔴 긴급`으로 자동 상향한다.
5. **정상 항목 포함 필수:** INFO 레벨 정상 완료 항목도 표에 포함하되 `⬜ 정상`으로 표시한다. 생략하면 "에러만 강조"되어 실제 처리 현황을 파악할 수 없다.
