# 2주차 계획

- 내가 만들 산출물: 에러 로그 추적 자동화 — 원본 로그 입력 → 에러 분류·빈도·근본원인·액션 아이템 보고서
- 실제 입력: 서버/앱 에러 로그 (타임스탬프·에러코드·스택트레이스·서비스명 포함)
- 이상적인 출력: 에러 종류별 분류표 + 심각도 + 재발 횟수 + 원인 추정 + 담당자 액션 항목
- AI가 맡을 일: 로그 파싱, 에러 분류, 패턴 감지, 중복 제거, 보고서 초안 작성
- 사람이 최종 확인할 일: 심각도 최종 판단, 핫픽스·배포 결정, 외부 알림(슬랙·이메일) 발송
- 오늘 자동화에서 제외할 일: 실제 서버 재시작, 코드 수정·배포, 슬랙·PagerDuty 직접 호출
