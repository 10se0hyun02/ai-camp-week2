# AI 완주반 2주차 — 에러 로그 추적 자동화

> 조서현 | 하네스 엔지니어링 주말반 | 2026-05-16

## 1. 이번 주 만드는 자동화 MVP

**에러 로그 추적 자동화** — 운영 서버 로그를 입력하면 에러 분류·빈도·근본원인·액션 아이템 보고서를 자동으로 생성한다.

## 2. AI vs 사람 역할 경계

| 담당 | 역할 |
|---|---|
| AI | 로그 파싱, 에러 분류·집계, 인과관계 추정, 보고서 초안 작성, ⚠️ 항목 플래그 |
| 사람 | 심각도 최종 판단, 핫픽스·배포 결정, 데이터 재처리 결정, 외부 알림(슬랙·이메일) 발송 |

## 3. 입력 예시 요약

- 운영 환경 에러 로그 (타임스탬프·에러코드·스택트레이스·서비스명 포함)
- 민감정보(실제 IP, 사용자 ID) 익명화 처리 완료
- 상세 예시: [`golden/input-example.md`](golden/input-example.md)

## 4. 이상적인 출력 예시 요약

- 에러 분류표 (ID·유형·서비스·빈도·심각도)
- 근본원인 추정 (인과관계 포함)
- 우선순위별 액션 아이템 (담당·기한)
- ⚠️ 사람 판단 필요 항목 분리
- 한 줄 요약
- 상세 예시: [`golden/output-example.md`](golden/output-example.md)

## 5. 첫 초안 vs 골든 출력 차이 (격차 분석)

| 기준 | 첫 초안 | 골든 출력 |
|---|---|---|
| 빠진 정보 | 발생 횟수·심각도·인과관계 없음 | 모두 포함 |
| 구조 | 산문 나열 | 5섹션 고정 구조 |
| 톤 | 추측성 구어체 | 로그 수치 근거 제시 |
| 사실 오류 | 근거 없는 해결책 제안 | 입력 기반만 |
| AI↔사람 경계 | ⚠️ 항목 표시 없음 | 명확히 분리 |

상세 분석: [`outputs/gap-analysis.md`](outputs/gap-analysis.md)

## 6. 3주차 제약 조건 후보

1. **출력 구조 고정:** [분류표 → 근본원인 → 액션 아이템 → ⚠️ 사람 검토 → 한 줄 요약] 5섹션 순서
2. **수치 원본 보존:** 발생 횟수·응답시간·버전은 로그 그대로, 반올림·생략 금지
3. **추측 표현 금지:** "아마", "것 같다" 대신 "원인 확정 불가 — 사람 확인 필요"
4. **사람 판단 항목 표시:** 배포 롤백·데이터 재처리·보안 위협은 반드시 ⚠️ 표시
5. **에러 ID 부여:** 각 에러에 E-NN / W-NN ID를 부여해 액션 아이템과 교차 추적 가능하게

## 7. 민감정보 처리

- 실제 서버 IP 주소: 포함 없음
- 실제 사용자 ID: `user-***-001` 형식으로 익명화
- 실제 내부 도메인·서비스명: 예시용으로 대체
- 실제 발송·결제·로그인·외부 시스템 조작: 과제 범위 외
